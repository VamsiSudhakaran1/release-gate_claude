What Does release-gate Actually Do?

Three simple checks. One powerful decision.

1

Initialize Your Governance Policy

Create a release-gate.yaml file that defines your safety requirements: request contracts, kill switches, ownership, fallback modes, and more.

↓
2

Run the Governance Check

Execute release-gate run --config release-gate.yaml and release-gate validates every policy you defined.

↓
3

Get Your Answer

release-gate returns PASS (deploy), WARN (review), or FAIL (fix). Use the exit code in your CI/CD pipeline.

What Gets Checked?

release-gate validates governance across multiple dimensions

📋

Request Contract

Verify your request format is well-defined and validated. Test both valid and invalid samples against your schema.

🔧

Operational Safeguards

Ensure kill switch, fallback mode, ownership, and runbook are all declared and documented. You're prepared for emergencies.

🔐

Safety Boundaries

Define identity requirements, access controls, rate limits, and budget enforcement. Control who can use your agent and how much it can spend.

🎯

Policy Enforcement

Convert your safety requirements from good intentions into actual checks. Policies that block bad deployments aren't just documented—they're enforced.

📊

Evidence Generation

Generate audit-ready reports that prove your governance is in place. Perfect for compliance teams and auditors.

🚫

Automated Gating

Integrate with CI/CD pipelines. Block bad deployments automatically. No manual review needed—the gate makes the decision.

Easy to Install. Easy to Use.

Up and running in 5 minutes. Zero external dependencies.

1

Install (One Command)

No complex setup. No external services. Just Python and two libraries.

pip install pyyaml jsonschema
2

Initialize (Template Generated)

release-gate creates a template configuration for you to customize.

python cli.py init --project my-agent
3

Customize (Your Policies)

Edit the generated YAML file with your governance requirements. Easy to read, easy to version control.

nano release-gate.yaml # Edit your policies
4

Run (Check Governance)

Execute the governance check. Get back PASS/WARN/FAIL. Use in CI/CD pipelines.

python cli.py run --config release-gate.yaml
5

Deploy (Automated or Manual)

If PASS, deploy automatically. If WARN, require review. If FAIL, block deployment. Your choice.

if [ $? -eq 0 ]; then deploy_to_production; fi
