The Problem: Uncontrolled AI Costs

❌ No Cost Limits

Agents can cost unlimited amounts. One mistake = $50K/day bill.

❌ No Validation

Configurations aren't validated before deployment. Broken agents go live.

❌ No Governance

No one checks if agents are safe. Kill switches don't exist.

❌ No Visibility

No one knows agent costs until the massive AWS bill arrives.

❌ No Enforcement

Configuration best practices are suggestions, not requirements.

❌ No Control

By the time you notice, damage is done and hard to fix.

The Solution: Governance Enforcement

💰 ACTION_BUDGET (Hero Check)

Validates cost limits. Prevents $50K mistakes. Full transparency in calculations.

  • Automatic cost estimation
  • Smart approval thresholds
  • Approval routing (Slack/Email)
  • Clear remediation steps
  • Dynamic pricing (any model)
  • Auto-detection from code

📋 INPUT_CONTRACT

Validates request schemas. Ensures valid/invalid samples pass/fail.

  • Schema validation
  • Sample testing
  • Type checking
  • Required fields

⏹ FALLBACK_DECLARED

Ensures kill switch exists. Validates safety measures are in place.

  • Kill switch validation
  • Fallback mode required
  • Ownership enforcement
  • Runbook requirement

🔐 IDENTITY_BOUNDARY

Enforces access control. Validates authentication and rate limits.

  • Authentication required
  • Rate limiting
  • Data isolation
  • Access control

Quick Start (5 Minutes)

Step 1: Install
$ pip install release-gate
Step 2: Create governance.yaml
project:
  name: my-agent

agent:
  model: gpt-4-turbo
  daily_requests: 500
  avg_input_tokens: 800
  avg_output_tokens: 400

checks:
  action_budget:
    max_daily_cost: 100
Step 3: Validate
$ release-gate check --config governance.yaml
Step 4: See Results
✅ All 4 checks passed. Deploy with confidence.
